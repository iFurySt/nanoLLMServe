# v0.6 Prefix Cache

## Goal

Reuse KV cache for shared prompt prefixes.

## Why

System prompts, few-shot examples, RAG prefixes, and agent loops repeat large prefixes.

## Scope

- prefix token hashing
- cache lookup
- matched prefix blocks
- reference counts
- LRU eviction

## Out Of Scope

- radix tree optimization unless needed for clarity
- distributed prefix cache
- persistence across process restarts

## Acceptance Criteria

- Repeated prefixes skip redundant prefill work.
- Cache hit and miss counts are visible.
- Benchmark shows before/after TTFT for repeated prefixes.

Current implementation statement:

- `PrefixCache` stores block-aligned prompt-prefix token hashes, sliced
  Hugging Face `past_key_values`, matched block ids, ref counts, hit/miss
  counters, and LRU eviction for unreferenced entries.
- `generate_one` and `stream_generate_one` can receive a `PrefixCache`; a
  strict shared prefix hit reuses cached KV and only pre-fills the uncached
  suffix before decode.
- This is an in-process single-request path feature. Static/continuous batch
  prefix reuse, distributed caches, persistent caches, and full exact-prompt
  logits reuse remain out of scope for this milestone.
