# v0.5 Block KV Cache Manager

## Goal

Manage KV cache in token blocks instead of one large contiguous tensor per request.

## Why

Block management introduces the main motivation behind PagedAttention: reduce KV cache fragmentation and improve memory use.

## Scope

- configurable block size, initially 16 or 32 tokens
- free block pool
- request-to-block table
- simple block allocation and release
- simplified paged KV model

## Out Of Scope

- full vLLM PagedAttention kernel
- GPU custom block kernels
- prefix cache

## Acceptance Criteria

- Requests allocate and release KV blocks.
- KV block usage is observable.
- Fragmentation behavior is demonstrated with a small benchmark.
