# v0.3 Static Batching

## Goal

Run multiple requests together in a fixed batch.

## Why

Static batching makes the batch dimension explicit before introducing continuous batching.

## Scope

- fixed batch queue
- padding
- attention masks
- batch decode loop
- whole-batch completion

## Out Of Scope

- mid-batch request admission
- per-request exit without rebuilding the batch

## Acceptance Criteria

- Multiple prompts can be processed in one model forward path.
- Batch size is visible in benchmark output.
- The limitations of static batching are documented.

Current limitation statement:

- The batch is fixed for the full decode trajectory; there is no mid-batch admission,
  preemption, request finishing-and-repacking, or shared work stealing in this milestone.
