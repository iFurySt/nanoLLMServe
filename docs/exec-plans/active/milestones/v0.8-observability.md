# v0.8 Observability

## Goal

Make engine behavior visible through metrics.

## Why

Serving optimizations are not meaningful if latency, throughput, queueing, and memory behavior are invisible.

## Scope

- TTFT
- TPOT
- throughput in tokens per second
- queue latency
- active batch size
- KV block usage
- Prometheus or a simple dashboard

## Out Of Scope

- full tracing stack
- hosted dashboards
- alerting

## Acceptance Criteria

- Runtime metrics are exposed.
- Benchmarks record the same metrics.
- Docs explain how to read the metrics.

## Progress
