# v0.2 KV Cache Decode

## Goal

Split prefill and decode so generation reuses KV cache.

## Why

KV cache is the line between naive repeated forward passes and real LLM inference.

## Scope

- `past_key_values`
- prefill/decode separation
- decode with only the last token
- TTFT and TPOT measurements

## Out Of Scope

- paged KV cache
- prefix cache
- custom attention kernels

## Acceptance Criteria

- Generation reuses KV cache after prefill.
- Benchmark reports TTFT and TPOT.
- A before/after comparison against v0.0 exists.
