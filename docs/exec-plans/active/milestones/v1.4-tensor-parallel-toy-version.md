# v1.4 Tensor Parallel Toy Version

## Goal

Demonstrate a minimal tensor parallel path.

## Why

Large model serving needs weight sharding. A toy implementation makes the core idea visible before production complexity.

## Scope

- split selected linear weights
- torch distributed setup
- all-reduce
- minimal NCCL path where available
- correctness comparison against a single-process baseline

## Out Of Scope

- full model coverage
- optimized communication
- pipeline parallelism
- expert parallelism

## Acceptance Criteria

- A toy sharded layer produces the same output as the unsharded baseline.
- A minimal model path can run with tensor parallel workers.
- Docs explain why TP differs from replica serving.
