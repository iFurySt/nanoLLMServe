# v1.3 Multi-GPU Minimal

## Goal

Run multiple worker replicas and route requests between them.

## Why

Data parallel serving is the simplest way to understand multi-GPU capacity before tensor parallelism.

## Scope

- N workers
- router
- round-robin routing
- least-loaded routing
- per-worker metrics

## Out Of Scope

- tensor parallelism
- cross-worker KV sharing
- autoscaling

## Acceptance Criteria

- Multiple workers can serve requests.
- Router policy can be switched.
- Per-worker load is observable.
